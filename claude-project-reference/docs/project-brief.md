# Projektbrief – KeasCare Markedssignaler

## Formål
KeasCare vil skabe et system, der automatisk finder relevante bosteder og salgsanledninger baseret på offentlige data og Monday CRM.

## Strategisk retning
Projektet skal ikke erstatte Monday. Monday skal fortsat bruges som CRM. KeasCare-systemet skal være intelligenslaget ovenpå.

## Kerneværdi
Systemet skal besvare:

- Hvilke bosteder findes der?
- Hvilke har nye tilsynsrapporter?
- Hvilke er ikke kunder i Monday?
- Hvilke bør KeasCare reagere på?

## Realistisk V1
Byg først en datadrevet dashboardprototype med mock-data og korrekt arkitektur.
